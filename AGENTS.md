## CSS Architecture

`css/main.css` uses CSS custom properties scoped to `:root`:

```css
:root {
  --green:        #1e4d2b;
  --green-dark:   #153820;
  --green-light:  #6aad32;   /* buttons, links, accents */
  --green-hover:  #5a9e2f;
}
```

Key layout classes and what they do:
- `.hero` + `.hero-inner` — full-height hero with two-column grid (text left, form right)
- `.booking-card` / `.booking-card-header` / `.booking-form` — the sticky sidebar estimate form
- `.service-grid` — 3-column service card grid (2-col on tablet, 1-col on mobile)
- `.split-section` — full-bleed 50/50 image + text layout (used for "Why Us")
- `.city-grid` + `.city-pill` — 5-column city link grid
- `.content-sidebar-layout` — 2-column layout for service and city detail pages (content + sticky form sidebar)
- `.review-grid` — 3-column testimonial grid
- `.stats-bar` + `.stats-grid` — dark green stat strip below hero
- `.area-grid` + `.area-card` — city cards on service areas hub
- `.features-grid` + `.feature-card` — 4-column feature highlights (about page)

## Context: Why This Approach

**Background:** Client wants 3 local Long Island service business websites modeled on the Neighborly franchise template (groundsguys.com, mrhandyman.com, mrrooter.com). The goal is to beat Neighborly's nationwide pages on Google by targeting specific Long Island cities with dedicated pages + LocalBusiness schema.

**Why clean CSS instead of the Neighborly CSS:** We extracted the 750KB Neighborly CSS but it requires their JS runtime (Tailwind JIT + React component tree) to render correctly. Without it, grid layouts break silently — especially the service card grids which were rendering as very narrow columns. The clean custom CSS approach is faster (15KB vs 750KB), fully predictable, and produces better visual results.

**SEO strategy:** Each of the 20 `/service-areas/[city]/` pages targets `"landscaping [city] NY"` with a LocalBusiness schema, city-specific h1/copy, and a working contact form. This beats Neighborly because Google proximity ranking favors locally-hosted, locally-specific pages over nationwide franchise location directories.
